# RoboMaster 队伍工作指南

这是一份用于队伍传承、协作管理和新人培养的工作指南。它不是一次性总结，而是下一届队员可以持续维护、复制执行、按赛季迭代的知识库。

## 使用方式

- `docs/`：正式指南章节，适合沉淀制度、流程和经验。
- `templates/`：可复制使用的表格和文档模板。
- `assets/`：图片、视频、附件等资料归档说明。
- `archive/`：历年总结、复盘和旧版本资料。

## 协作规则

1. 每个章节指定一名主负责人，其他成员可以补充。
2. 修改制度、流程、模板时，必须在文档末尾记录更新时间和修改人。
3. 经验复盘要写清楚现象、原因、解决方案和下次避免方法。
4. 不确定的内容先标记为 `待确认`，不要用模糊说法代替流程。
5. 飞书用于多人协作编辑，Git/GitHub 用于正式版本归档。

## 协作分工

为避免多人协作时内容重复、遗漏或互相等待，每个模块都需要指定主负责人。主负责人负责组织填写、收集资料、统一口径和最终验收，不代表所有内容都由主负责人一个人完成。

| 模块 | 主负责人 | 协作人 | 必须交付 |
| --- | --- | --- | --- |
| 指南使用说明、版本管理 | 队长/副队长 | 各组组长 | 目录规则、维护规则、版本记录 |
| 团队治理与协作 | 队长/副队长 | 各组组长、兵种负责人 | 组织架构、职责边界、任务死锁处理机制 |
| 会议制度与任务管理 | 队长/副队长 | 各组组长 | 会议纪律、任务状态、延期处理、飞书看板规则 |
| 组长工作手册 | 队长/副队长 | 机械/电控/算法/运营组长 | 组长职责、设计把关、工作验收标准 |
| 技术研发流程 | 技术负责人/副队长 | 机械、电控、算法组长 | 研发流程、评审流程、测试验收流程 |
| 机械相关内容 | 机械组长 | 各兵种机械负责人 | 机械设计、加工装配、维修、规则冗余经验 |
| 电控相关内容 | 电控组长 | 各兵种电控负责人 | 接线、板卡、固件、调参、联调经验 |
| 算法相关内容 | 算法组长 | 自瞄/导航负责人 | 环境配置、模型、相机、部署、调试经验 |
| 文档与资料沉淀 | 文档负责人/副队长 | 各模块负责人 | 命名规范、归档规范、GitHub/飞书资料链接 |
| 比赛材料与视频拍摄 | 队长/运营负责人 | 兵种负责人、操作手、摄像 | 视频流程、拍摄检查、提交记录 |
| 采购、预算、报销 | 运营/财务负责人 | 各组采购发起人 | 飞书审批规则、发票要求、报销流程、物资入库 |
| 赛前、赛中、赛后流程 | 队长/备场负责人 | 操作手、兵种负责人、物资负责人 | 赛前检查表、上场前检查、赛后复盘 |
| 招新、培训与考核 | 培训负责人 | 各组组长、正式队员 | 培训路线、作业、考核标准 |
| 传承与交接 | 队长/副队长 | 各岗位现任负责人 | 队长、组长、兵种负责人交接清单 |
| 事故复盘与经验库 | 队长/技术负责人 | 事故相关负责人 | 问题现象、原因、解决方案、预防措施 |

## 填写验收规则

每个章节在交付前至少满足：

- 有明确负责人和更新时间。
- 流程类内容能让新人照着执行。
- 经验类内容写清楚具体场景，不能只写结论。
- 涉及飞书、GitHub、网盘、代码仓库的内容要附链接或说明位置。
- 模板类内容能被下一届直接复制使用。
