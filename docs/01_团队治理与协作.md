# 01_团队治理与协作

## 核心原则

所有设计、任务和协作都应服务于实际问题。不能只追求形式上的完成，而要确保任务最终可以交付、调试、维护、拍摄和比赛使用。

## 队伍组织架构

待补充：填写本队当前组织架构，包括队长、副队长、各组组长、兵种负责人、运营负责人等。

## 队长职责

- 制定赛季整体目标和时间节点。
- 协调跨组资源和冲突。
- 发现重大延期后及时重新分配任务。
- 确保报名、规则、材料、视频等关键事项按期完成。
- 推动赛季复盘和传承文档完成。

## 组长职责

- 拆解本组任务。
- 明确负责人、协作人、截止时间和验收标准。
- 检查组员设计或实现的合理性、必要性和风险。
- 发现阻塞后及时协调。
- 维护本组文档和资料归档。

## 兵种负责人职责

- 对该兵种从设计、装配、调试到视频拍摄负责。
- 组织机械、电控、算法联合评审。
- 在联调和考核视频拍摄时保证相关负责人到场。
- 记录该兵种问题、改进建议和下一届注意事项。

## 跨组协作原则

1. 新车设计前必须进行机械、电控、算法联合沟通。
2. 机械方案需要提前考虑走线、主控、分电板、电机选型、电机朝向和维修空间。
3. 电控和算法需求不能等机械全部完成后才提出。
4. 自瞄、导航等系统第一次调试时，兵种机械、电控、算法负责人应同时在场。
5. 任何设计都应说明它给其他组带来的额外难度。

## 任务死锁处理机制

当任务出现“我在等他，他在等我”的情况时，必须立即处理。

| 阻塞时间 | 处理方式 |
| --- | --- |
| 当天 | 负责人在任务群或飞书任务中说明阻塞原因 |
| 超过 24 小时 | 组长介入，明确下一步动作 |
| 超过 48 小时 | 队长介入，必要时重新分配任务 |

## 消极、失联和延期处理

根据队伍实际情况制定提醒、约谈、替换负责人、取消资格等处理方式。

## 上一届答疑机制

- 暑假前优先线下答疑。
- 离校后保留线上联系方式。
- 每个关键方向至少指定一名上一届答疑人。
- 答疑内容应整理回文档，不只停留在聊天记录。
