# 02_会议制度与任务管理

## 会议类型

| 会议 | 频率 | 参与人 | 目标 |
| --- | --- | --- | --- |
| 队伍周会 | 每周一次 | 队长、组长、核心成员 | 同步进度、暴露风险、确定本周重点 |
| 组内例会 | 每周一次或按需 | 组长、组员 | 拆任务、查阻塞、做验收 |
| 专项会议 | 按需 | 相关负责人 | 解决具体问题，如联调、视频、采购 |
| 赛前会议 | 赛前固定安排 | 全体相关人员 | 检查物资、人员、机器人状态和比赛流程 |

## 会议纪律

- 会前必须准备进度和问题。
- 会议只讨论需要多人决策或同步的事项。
- 每个任务必须留下负责人、截止时间和验收标准。
- 会议结束后当天发布会议纪要。
- 无故缺席、迟到、未准备的问题应由组长记录并处理。

## 任务状态定义

| 状态 | 含义 |
| --- | --- |
| 未开始 | 已分配但尚未行动 |
| 进行中 | 正在推进 |
| 阻塞 | 需要他人、物资、场地或决策支持 |
| 待验收 | 负责人认为完成，等待组长或相关人员检查 |
| 已完成 | 符合验收标准并归档 |

## 任务记录字段

每个任务至少包含：

```text
任务名称：
负责人：
协作人：
截止时间：
当前状态：
验收标准：
阻塞问题：
相关资料链接：
```

## 延期处理

1. 预计无法按期完成时，负责人必须提前说明。
2. 组长判断是否调整时间、增加人手或降低范围。
3. 关键路径任务延期时，必须同步队长。
4. 多次延期且无有效进展时，应重新分配负责人。

## 飞书任务板建议

建议按照 `未开始 / 进行中 / 阻塞 / 待验收 / 已完成` 建立看板。每周会前由负责人更新状态。
