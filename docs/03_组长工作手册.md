# 03_组长工作手册

## 组长的核心工作

组长不是只负责催进度，而是负责让本组任务能够被拆解、执行、验收和传承。

## 任务拆解

每个任务应拆到可以被一个人明确推进的粒度。避免出现“优化自瞄”“做好英雄”这类无法验收的任务。

## 设计把关

组长应要求组员在设计或方案提交时说明：

```text
设计目的：
解决的实际问题：
为什么需要这个设计：
替代方案：
主要缺点：
对其他组的影响：
加工、装配、维修难度：
调试和拍摄风险：
验收标准：
```

## 工作验收

验收不能只看“做了没有”，而要看是否满足使用场景。

| 类型 | 验收重点 |
| --- | --- |
| 机械 | 强度、重量、尺寸、装配、维修、规则冗余 |
| 电控 | 接线、稳定性、保护、日志、可调试性 |
| 算法 | 环境配置、可复现、识别效果、部署性能 |
| 运营 | 材料完整、时间节点、格式要求、归档 |

## 风险发现

组长每周应主动检查：

- 是否有任务被阻塞。
- 是否有人长期没有实际产出。
- 是否有设计没有经过跨组评审。
- 是否有关键物资没有采购。
- 是否有规则更新影响当前方案。

## 文档要求

本组每个关键项目都应留下：

- 方案文档
- 测试记录
- 问题记录
- 最终文件链接
- 下一届建议
