# 04_技术研发流程

## 总流程

```text
需求提出 -> 方案设计 -> 跨组评审 -> 打样/开发 -> 装配 -> 联调 -> 测试 -> 验收 -> 归档 -> 复盘
```

## 需求提出

需求应来自比赛规则、机器人任务、历史问题或明确的性能目标。不要为了炫技而增加不必要复杂度。

## 方案设计

方案文档至少包含：

- 背景和目标
- 设计思路
- 关键参数
- 替代方案比较
- 风险和缺点
- 对其他组的影响
- 时间计划
- 验收标准

## 跨组评审

新车、新机构、新控制方案、新算法部署前，应组织相关组评审。

评审重点：

- 是否符合规则尺寸、重量、安全要求。
- 是否方便走线、安装主控、分电板、相机、雷达等设备。
- 电机、相机、小电脑等硬件选型是否匹配需求。
- 是否便于调试、维修、运输和视频拍摄。
- 是否给其他组带来不可接受的难度。

## 联调要求

- 第一次自瞄调试时，兵种机械、电控、算法负责人应在场。
- 第一次导航调试时，底盘、电控、算法负责人应在场。
- 发现说不清的问题时，应补充日志、视频或复现步骤。
- 联调结果必须记录，不要只停留在聊天记录。

## 测试与验收

测试应尽量接近比赛场景。验收前要明确：

```text
测试项目：
测试环境：
测试步骤：
通过标准：
失败现象：
改进措施：
负责人：
```

## 当前重点经验

- 自瞄和导航是弱队与强队拉开差距的关键技术，应提前投入硬件和调试时间。
- 英雄拨弹盘和弹链设计必须重点验证卡弹、弹速稳定和单发控制。
- 哨兵应重点检查敌我识别、自瞄索敌、云台坐标转换和电气稳定性。
- 规则更新后必须重新检查尺寸、重量和制作规范，避免投入打水漂。
