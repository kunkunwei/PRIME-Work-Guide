# 05_文档与知识沉淀规范

## 文档目标

文档不是为了留痕，而是为了让下一位接手的人能更快复现、维护和改进。

## 必须沉淀的内容

- 机械设计方案和图纸说明
- 电控接线、板卡、固件、调参记录
- 算法环境配置、模型、数据集、部署步骤
- 采购论证和物资清单
- 视频拍摄计划和最终材料
- 比赛复盘和事故记录
- 队长、组长、兵种负责人交接文档

## 文档基本结构

```text
标题：
作者：
最后更新时间：
适用赛季：
适用对象：
背景：
操作步骤：
常见问题：
风险提醒：
相关资料链接：
维护人：
```

## 命名规范

建议使用：

```text
赛季_模块_主题_版本
```

示例：

```text
2026_英雄_拨弹盘设计复盘_v1
2026_自瞄_环境配置说明_v1
2026_赛前_物资清单_v1
```

## 归档原则

- 飞书适合协作编辑。
- GitHub 适合正式归档和版本追踪。
- 原始资料不要只存在个人电脑。
- 重要资料应有至少一份云端备份。
- 聊天记录中的关键结论应整理成文档。
