# 11_事故复盘与经验库

## 复盘原则

复盘不用于追责，而用于让队伍下一次不再付出同样代价。每个事故都应写清楚现象、原因、解决方法和预防措施。

## 复盘模板

```text
问题名称：
发生时间：
涉及兵种/模块：
现象：
直接原因：
根本原因：
当时怎么解决：
下次如何避免：
相关负责人：
相关资料：
```

## 已知问题条目

### 英雄拨弹盘和弹链卡弹

待补充：现象、具体原因、测试条件、改进方案。

### 哨兵打自己人

待补充：红蓝方设置、自瞄索敌、敌我识别、上场前检查流程。

### 自瞄相机成像异常

待补充：相机、镜头、曝光、颜色识别、距离表现、硬件选型建议。

### 导航小电脑性能不足或不稳定

待补充：硬件型号、内存/显存占用、进水/损坏情况、替代方案。

### 视频拍摄进程被阻塞

待补充：阻塞原因、人员安排问题、拍摄顺序优化。

### 任务互相等待导致死锁

待补充：任务链条、负责人、组长介入时机和后续制度。
